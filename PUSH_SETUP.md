# 締切プッシュ通知のセットアップ手順

リクエスト締切の **3日前** と **前日** に、スマホへ通知（プッシュ通知）を送るための設定です。
外部の有料サービスは不要で、Google Apps Script（GAS）だけで動きます。

---

## 全体の流れ

```
スマホ（ホーム画面に追加したアプリ）         GitHub Pages（index.html / sw.js）
   ┌─ 「🔔 締切通知」ボタンで購読 ──────────►  購読情報を GAS へ送信
   └─ 締切日が変わると自動で GAS へ送信
                                              Google Apps Script（apps-script-push.gs）
                                                ├─ 購読情報・締切日を保存
                                                └─ 毎日1回、3日前/前日なら通知を送信 ─► スマホに通知
```

---

## 1. Apps Script を作る

1. <https://script.google.com> を開き、**新しいプロジェクト** を作成。
2. リポジトリの **`apps-script-push.gs`** の中身を全部コピーして貼り付け（既存のコードは消してOK）。
3. 上部の設定（`PUSH_TOKEN` / `VAPID_PUBLIC` / `VAPID_PRIVATE` / `VAPID_SUBJECT`）はそのままで動きます。
   - `VAPID_SUBJECT` は連絡先メール。必要なら自分のアドレスに変更可。
4. プロジェクトの **タイムゾーン** を日本にします：
   左の「⚙ プロジェクトの設定」→「タイムゾーン」を **(GMT+09:00) 東京** に。

## 2. ウェブアプリとしてデプロイ

1. 右上 **デプロイ → 新しいデプロイ**。
2. 種類：**ウェブアプリ**。
3. 「次のユーザーとして実行」＝ **自分**、「アクセスできるユーザー」＝ **全員**。
4. **デプロイ** → 表示される **ウェブアプリの URL**（`https://script.google.com/macros/s/……/exec`）をコピー。

## 3. index.html に URL を貼る

`index.html` の上の方にある2行を編集します（暗号化されていない部分です）。

```js
const PUSH_URL="";                       // ← ここに 手順2 でコピーした URL を貼る
const PUSH_TOKEN="bihin-push-7f3a9c2e";  // ← Apps Script の PUSH_TOKEN と同じ値（変えるなら両方）
```

例：
```js
const PUSH_URL="https://script.google.com/macros/s/AKfycb..../exec";
```

変更を **commit & push** して、GitHub Pages に反映させます。

## 4. 毎日実行するトリガーを設定

1. Apps Script の左メニュー **⏰ トリガー** → **トリガーを追加**。
2. 実行する関数：**`sendDeadlineNotifications`**
3. イベントのソース：**時間主導型** → **日付ベースのタイマー** → 例「午前9時～10時」。
4. 保存。

これで毎日1回、締切の3日前と前日に自動で通知が送られます。

## 5. スマホ側（各自）

- **iPhone**：Safari でサイトを開く → 共有 → **ホーム画面に追加** → そのアイコンから起動 →
  右下の **「🔔 締切通知」** をタップして許可。※ **iOS 16.4 以上** が必要です。
- **Android（Chrome）**：サイトを開く → メニュー → **アプリをインストール / ホーム画面に追加** →
  **「🔔 締切通知」** をタップして許可。

ボタンが緑色の「🔔 通知オン」になれば購読完了です。

---

## 動作テスト

すぐに通知が届くか試すには、Apps Script のエディタで一時的に次を実行します（自分の端末で購読済みのとき）：

```js
function testNow(){ Logger.log(sendToAll_("テスト通知","届けばOKです")); }
```

`testNow` を実行 → スマホに「テスト通知」が来れば成功です。

締切判定のテストは、`sendDeadlineNotifications` を手動実行しても、その日が3日前/前日でなければ送られません（仕様）。

## よくある質問

- **通知が来ない**
  - スマホがホーム画面アイコンから起動されているか（特に iPhone）。
  - 端末の設定でアプリの通知が許可されているか。
  - `PUSH_URL` が正しく入っているか、トリガーが設定されているか。
  - Apps Script の「実行数」ログにエラーが出ていないか。
- **締切日はどこから来る？** アプリの「リクエスト」画面で設定した「リクエスト締切日」です。誰かがアプリを開くと自動的に GAS に伝わります。
- **何人でも使える？** 購読した端末すべてに送られます。失効した購読は自動的に削除されます。
- **VAPID キーを作り直したい？** 新しい鍵ペアを作って `apps-script-push.gs` と `index.html`（`VAPID_PUBLIC`）の両方を更新し、各自に再購読してもらってください。

> 技術メモ：このバックエンドは VAPID(ES256) 署名と aes128gcm 本文暗号（RFC 8291/8292）を Apps Script だけで実装しています（暗号処理は Node.js で正当性を検証済み）。
